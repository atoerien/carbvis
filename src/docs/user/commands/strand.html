Command: strand

Usage:
strand [ atom-spec ] [ replace  true | false ] [ update  true | false ] [ maxRingSize  N ] [ maxPathLen  length ] [ radius  r ] [ color  linkage-colormap ] [ candyCane  true | false ] [ sphereRadius  r-sphere ] [ sphereColor  ring-colormap ]

The strand command creates a Strand model for the specified atoms, inspired by the tube representation used in protein ribbon diagrams. Each glycosidic linkage is drawn as a smooth cylindrical tube whose color encodes the linkage dihedral angles, making unusual glycosidic torsion angles easy to spot while still allowing the overall backbone shape to be followed. Tubes are rendered along smooth curves that pass through the ring centroids and linkage atoms, producing a continuous path through the molecule.

An optional sphere at each ring centroid can be added with sphereRadius to make individual linkage boundaries explicit and to provide visual anchors for the start and end of each backbone segment.

Strand emphasizes overall backbone shape and torsion angles. For ring puckering and absolute ring orientation, see paperchain. For backbone twist and relative ring orientation, see twister.

If atom-spec is omitted, all atoms in all open atomic structures are used.

See also: CarbVis, twister, paperchain

replace  true | false
Whether to remove existing Strand models that overlap the new one (default true).
update  true | false
Whether to automatically update the model if the structure changes (default true).
maxRingSize  N
Maximum ring size considered when identifying carbohydrate rings (default 10 atoms).
maxPathLen  length
The maximum path length in bonds to search when finding linkages between rings (default 5).
radius  r
The radius of the tubes in Å (default 0.75).
color  linkage-colormap
The linkage colormap used to color the tubes. Defaults to the default colormap.
candyCane  true | false
Enables the "Candy Cane" rendering variant (default false). The tube cross-section is split into alternating colored and white sectors, producing a striped appearance that follows the backbone twist. Spheres (if enabled) are not striped.
sphereRadius  r-sphere
Radius of the sphere drawn at the centroid of each sugar ring, in Å. When omitted, no discrete sphere is added; instead the tube color is blended smoothly through each ring junction. Set sphereRadius to a value larger than radius to render an explicit sphere at every ring.
sphereColor  ring-colormap
The ring colormap used to color the ring spheres. When omitted, the spheres inherit their color from the adjacent tube segments.
